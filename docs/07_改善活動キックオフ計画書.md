## 1. 目的

本書は、既存の仕様書（docs/01-06）を「実行計画」に変換し、改善活動を開始するための運用基準を定める。  
特に、公開ブロッカー（P0）の早期解消と、KGIに直結する導線（予約・取材問い合わせ）の実装を最優先とする。

## 3. 成果目標（公開前に必達）

### 必達KGI連動項目

1. レストラン来店率向上に向けた予約導線の明確化
2. マスコミ取材エンゲージメント向上に向けた Press / Contact 導線の整備

### 公開判定（Definition of Done）

- [ ] 13ページ構成とURLスラッグが仕様どおり
- [ ] 全主要ページで title / meta description / OGP 設定済み
- [ ] TOP、Restaurant、各レストラン個別、Press に2次CTAが設置済み
- [ ] Press ページ（フォーム・資料DL導線）が稼働
- [ ] GA4 / Search Console が接続済みで主要イベント計測可能

## 4. 初動14日ロードマップ

### Week 1: 公開基盤の確立（P0中心）

### Day 1-2: 技術基盤と計測基盤

- [ ] 独自ドメイン取得・Wix接続・SSL確認
- [ ] Wixプレミアムプラン加入（広告バナー非表示確認）
- [ ] GA4プロパティ作成、測定ID接続、リアルタイム確認
- [ ] Search Console 所有権確認、サイトマップ送信

### Day 3-4: ページ構造とSEO基盤

- [ ] 13ページ構造を作成、親子関係（Restaurant/Story配下）を設定
- [ ] グローバルナビ・フッターナビ反映
- [ ] 全ページのURLスラッグを英語化
- [ ] title / meta description を仕様どおり設定
- [ ] OGP画像（TOP用・共通用）を作成し適用

### Day 5-7: コンバージョン導線の初期実装

- [ ] Press / Contact ページ新設（ヒーロー、フォーム、運営者情報）
- [ ] TOPヒーローに2段階CTA（知る/レストラン）を配置
- [ ] TOP下部、About、Sustainabilityに Press CTA を配置
- [ ] Restaurant 一覧カードから個別ページ導線を明確化

### Week 2: KGI直結の強化（P1中心）

### Day 8-10: レストラン導線強化

- [ ] レストラン個別ページ4本の雛形を作成
- [ ] 予約方法に応じたCTA（外部予約 / tel / 未開店案内）を実装
- [ ] Google Map、営業時間、定休日など必須情報を実装

### Day 11-12: Press強化と素材整備

- [ ] プレスキット（PDF/ZIP/ロゴ）をアップロードしDL導線を実装
- [ ] メディア掲載実績セクション（最低1件）を公開
- [ ] フォーム送信通知メール、完了メッセージ確認

### Day 13-14: QAと公開判定

- [ ] モバイル表示確認（CTAタップ領域、改行、余白）
- [ ] 全ページリンク切れ確認
- [ ] GA4イベント送信テスト（予約、問い合わせ、DL、電話）
- [ ] 公開判定チェック実施

## 5. 作業分担（暫定）

| 役割 | 主担当作業 | 補足 |
|---|---|---|
| PM/ディレクション | 進行管理、優先順位調整、公開判定 | 週次レビュー実施 |
| Wix実装担当 | ページ作成、CTA配置、フォーム実装 | モバイル最適化含む |
| コンテンツ担当 | テキスト、写真、プレスキット準備 | レストラン情報回収を含む |
| 計測担当 | GA4/SC設定、イベント検証、週次レポート | UTM運用ルール管理 |

## 7. 週次レビュー運用

毎週1回、以下を確認する。

### 進捗

- 完了タスク数（P0/P1）
- ブロッカー数と解消予定日

### 成果指標（GA4）

- Restaurantページ閲覧数
- 予約CTAクリック数（click_reservation）
- 問い合わせ送信数（submit_contact）
- プレスキットDL数（download_presskit）
