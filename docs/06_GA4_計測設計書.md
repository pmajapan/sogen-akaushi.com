# GA4 / Search Console 設定・計測設計書（草原あか牛HP）

## 1. Google Analytics 4（GA4）の設定

### 初期セットアップ手順

1. **GA4 プロパティ作成**
   - Google Analytics（https://analytics.google.com/）にログイン
   - 「管理」>「プロパティを作成」
   - プロパティ名: `草原あか牛HP`
   - タイムゾーン: `日本`、通貨: `日本円（JPY）`

2. **データストリーム作成**
   - 「ウェブ」を選択
   - ウェブサイトの URL: `https://yourdomain.com`（独自ドメイン）
   - ストリーム名: `草原あか牛HP - Web`
   - 測定 ID（G-XXXXXXXXXX）をメモ

3. **Wix への接続**
   - Wix ダッシュボード > 「マーケティング・SEO」 > 「マーケティング統合」
   - 「Google アナリティクス」をクリック
   - 測定 ID（G-XXXXXXXXXX）を入力して接続
   - または: ダッシュボード > 「設定」 > 「カスタムコード」 > ヘッドに GA4 タグを貼り付け

4. **リアルタイムレポートで動作確認**
   - サイトにアクセスした状態で GA4 の「リアルタイム」を確認
   - ユーザーが計測されていることを確認

## 2. KGI に連動したイベント計測設計

### KGI と計測指標の対応

| KGI | 計測すべき指標 | GA4 イベント |
|-----|-------------|------------|
| レストラン来店率向上 | 予約ボタンクリック数、店舗情報閲覧数 | `click_reservation`, `view_restaurant` |
| マスコミ取材エンゲージメント向上 | プレスキットDL数、問い合わせ送信数 | `download_presskit`, `submit_contact` |

### イベント一覧

| イベント名 | トリガー | パラメータ | 種別 |
|-----------|---------|----------|------|
| `click_reservation` | レストラン予約ボタンクリック | `restaurant_name`, `location` | カスタムイベント |
| `view_restaurant` | レストラン個別ページ表示 | `restaurant_name`, `chef_name` | 自動（ページビュー） |
| `download_presskit` | プレスキットダウンロードクリック | `file_type` | カスタムイベント |
| `submit_contact` | お問い合わせフォーム送信完了 | `inquiry_type` | カスタムイベント |
| `click_phone` | 電話番号リンクのクリック | `restaurant_name` | カスタムイベント |
| `click_external_link` | 外部サイト（レストラン公式等）への遷移 | `destination_url`, `link_text` | 自動（拡張計測） |
| `scroll` | ページの90%スクロール | `page_path` | 自動（拡張計測） |
| `share` | SNSシェアボタンクリック | `page_path`, `platform` | カスタムイベント |

### Wix でのカスタムイベント実装方法

Wix ではカスタムコード（Velo）またはカスタムHTMLを使ってカスタムイベントを送信できる。

**方法A: Wix の「カスタムコード」機能**

ダッシュボード > 設定 > カスタムコード > ボディ末尾に追加:

```html
<script>
document.addEventListener('click', function(e) {
  var target = e.target.closest('a[href*="reservation"], a[href*="reserve"], a[href*="booking"]');
  if (target) {
    gtag('event', 'click_reservation', {
      'restaurant_name': target.dataset.restaurant || target.textContent.trim(),
      'location': target.dataset.location || ''
    });
  }
});
</script>
```

**方法B: GA4 の拡張計測を活用**

GA4 管理画面で拡張計測を有効化（デフォルトON）:
- [x] ページビュー
- [x] スクロール
- [x] 離脱クリック（外部リンク）
- [x] サイト内検索
- [x] フォーム操作

これだけで以下が自動計測される:
- ページビュー（各レストランページの閲覧数）
- 外部リンクのクリック（予約サイトへの遷移）
- スクロール深度（90%到達率）

## 3. コンバージョン設定

GA4 で以下をコンバージョン（キーイベント）として登録:

| コンバージョン名 | 対応イベント | 優先度 |
|---------------|-----------|-------|
| レストラン予約クリック | `click_reservation` | 最高 |
| お問い合わせ送信 | `submit_contact` | 最高 |
| プレスキットDL | `download_presskit` | 高 |
| 電話発信クリック | `click_phone` | 高 |

**GA4 での設定手順**:
1. GA4 管理画面 > 「イベント」
2. 対象イベントの「キーイベントとしてマーク」をON

## 4. レポート・ダッシュボード設計

### 週次確認指標

| 指標 | 確認方法 | 目標の目安 |
|------|---------|----------|
| サイト全体セッション数 | GA4 > レポート > ユーザー獲得 | 公開1ヶ月: 月500セッション以上 |
| Restaurant ページ閲覧数 | GA4 > レポート > ページとスクリーン | 全セッションの30%以上 |
| 予約ボタンクリック率 | コンバージョン数 / Restaurant PV | 5%以上 |
| お問い合わせ送信数 | コンバージョンレポート | 月2件以上 |
| プレスキットDL数 | コンバージョンレポート | 月5件以上 |
| 平均セッション時間 | GA4 > レポート > エンゲージメント | 2分以上 |
| 直帰率 | GA4 > レポート > エンゲージメント | 60%以下 |

### GA4 探索レポート（推奨作成）

1. **ユーザーフロー分析**: TOP → Restaurant → 個別レストラン → 予約クリック の流れを可視化
2. **流入元別コンバージョン**: どのチャネルから来たユーザーが予約に至るか
3. **ページ別エンゲージメント**: 各ページの滞在時間・スクロール率を比較

## 5. Google Search Console の設定

### セットアップ手順

1. **プロパティ追加**
   - Google Search Console（https://search.google.com/search-console/）にアクセス
   - 「プロパティを追加」
   - 「URLプレフィックス」を選択し `https://yourdomain.com` を入力

2. **所有権の確認（Wix）**
   - HTML タグ方式が簡単:
     - Search Console から `<meta name="google-site-verification" content="..." />` をコピー
     - Wix ダッシュボード > 「マーケティング・SEO」 > 「SEOツール」 > 「サイト認証」
     - Google のタブで認証コードを貼り付け

3. **サイトマップ送信**
   - Search Console > 「サイトマップ」
   - `https://yourdomain.com/sitemap.xml` を送信
   - Wixは自動でサイトマップを生成・更新する

### 定期確認項目

| 確認項目 | 頻度 | 確認場所 |
|---------|------|---------|
| インデックス状況 | 週1回 | ページ > インデックス登録結果 |
| 検索クエリ | 週1回 | 検索パフォーマンス |
| クロールエラー | 月1回 | ページ > ページがインデックスに登録されなかった理由 |
| モバイルユーザビリティ | 月1回 | エクスペリエンス > モバイルユーザビリティ |

### 公開直後の対応

1. 全ページの URL を「URL検査」で個別にインデックス登録をリクエスト
2. 特に以下を優先:
   - TOP（`/`）
   - Restaurant 一覧（`/restaurant`）
   - 各レストラン個別ページ
   - Press（`/press`）

## 6. UTMパラメータ設計（SNS・メール配信用）

外部からの流入を正確に計測するため、URLにUTMパラメータを付与:

| 用途 | utm_source | utm_medium | utm_campaign | URL例 |
|------|-----------|-----------|-------------|------|
| SNS投稿（Instagram） | `instagram` | `social` | `launch_202603` | `?utm_source=instagram&utm_medium=social&utm_campaign=launch_202603` |
| SNS投稿（Facebook） | `facebook` | `social` | `launch_202603` | `?utm_source=facebook&utm_medium=social&utm_campaign=launch_202603` |
| プレスリリース | `press_release` | `referral` | `launch_202603` | `?utm_source=press_release&utm_medium=referral&utm_campaign=launch_202603` |
| メール配信 | `email` | `email` | `newsletter_01` | `?utm_source=email&utm_medium=email&utm_campaign=newsletter_01` |

Google URL Builder（https://ga-dev-tools.google/campaign-url-builder/）を使用して生成。
