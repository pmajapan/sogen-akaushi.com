# レストラン個別ページ仕様書（草原あか牛HP）

## 個別ページ化の推奨理由

| 観点 | アンカーリンク方式（現状） | 個別ページ方式（推奨） |
|------|---------------------|-------------------|
| SEO | 1ページに全店舗が集約。店舗名・シェフ名での検索流入が取りにくい | 各店舗が独立URLを持ち、店舗名・シェフ名・地域名での検索流入が狙える |
| SNSシェア | シェア時に全店舗まとめのOGPになる | 各店舗の写真・紹介で個別にシェアできる |
| 予約導線 | アンカーリンク先が長いページの一部で離脱しやすい | 専用ページ内で予約情報まで完結する |
| ページ滞在時間 | Restaurant ページ全体のスクロール量が膨大 | 興味のある店舗だけを深く読める |
| 運用負荷 | やや低い | ページ数が増えるが、テンプレート化で軽減可能 |

**結論**: KGI「レストラン来店率向上」の達成には個別ページ化が効果的。

## 予約導線の設計

各店舗の予約方法に応じた CTA を設置:

| 予約方法 | CTA実装 |
|---------|--------|
| 外部予約サイト（食べログ・一休等） | 外部リンクボタン（新規タブで開く） |
| 電話予約のみ | 電話番号を大きく表示 + `tel:` リンク（モバイル対応） |
| 公式サイト予約フォーム | 外部リンクボタン |
| 予約不可（Tradeoff Diner 未開店等） | 「オープン情報を受け取る」→ Press ページのフォームへ |

**注意**: 各レストランに予約方法・リンク先を確認する必要あり。

## Wix での実装手順

### 個別ページの作成

1. エディタ > 「ページ・メニュー」 > 「+ ページを追加」
2. 「空白ページ」を選択
3. ページ名を店舗名に設定
4. ページを Restaurant の子ページとしてドラッグ＆ドロップ
5. SEO設定でスラッグ・タイトル・meta description を設定
6. 上記テンプレートに沿ってセクションを構築

### Google Map 埋め込み

1. 「+ 追加」>「埋め込みコード」>「地図」
2. Google Maps の埋め込みコードを取得して貼り付け
3. または Wix の「Google マップ」アプリを使用

### テンプレートの流用

1. 1ページ目を丁寧に作り込む
2. 「ページ・メニュー」からそのページを「複製」
3. 複製したページのテキスト・画像・リンクを差し替え
4. これを4店舗分繰り返す
